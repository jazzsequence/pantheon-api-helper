# pantheon-api-helper — Claude Instructions

This is the source package for `pantheon-api-helper`. It ships scripts that
fetch the Pantheon API Swagger spec and generate structured markdown docs for
AI agents to consume.

## Project structure

```
scripts/
  fetch-spec.js   Downloads swagger.json → .cache/swagger.json
  generate.js     Converts spec → docs/ (markdown, digest-based)
  postinstall.js  Runs on npm install: fetch → generate → copy → patch AGENTS.md/.gitignore
  cli.js          npx pantheon-api-helper <update|generate|fetch>
src/
  index.js        Programmatic API (docsDir, rootDigest, section)
spec/             Gitignored — fetched at install time
.cache/           Gitignored — swagger.json cached here by fetch-spec.js
docs/             Gitignored — generated by generate.js, copied to .pantheonapi-docs/
```

## Development commands

```bash
# Fetch latest spec from api.pantheon.io
node scripts/fetch-spec.js

# Regenerate docs from cached spec
node scripts/generate.js

# Full cycle: fetch + generate + install to a test project
node scripts/postinstall.js   # run from a consuming project's root

# CLI
node scripts/cli.js update
node scripts/cli.js generate
node scripts/cli.js fetch
```

## What NOT to commit

- `spec/` — fetched at install time
- `docs/` — generated at install time
- `.cache/` — intermediate cache
- `.claude/` — local Claude Code settings

Only commit: `scripts/`, `src/`, `AGENTS.md`, `CLAUDE.md`, `README.md`, `package.json`, `.gitignore`

## Modifying the doc generator

The generator lives in `scripts/generate.js`. Key areas:

- **`SITES_GROUPS`** (top of file) — regex rules for splitting the 66 sites endpoints into sub-sections. Add a new entry here to create a new sub-section file.
- **`renderEndpoint()`** — controls per-endpoint markdown output (summary, params, return type)
- **`renderDefinition()`** — controls schema table output
- **`renderResponse()`** — resolves Swagger 2.0 `$ref` responses through the top-level `responses` map

After editing the generator, re-run:
```bash
node scripts/fetch-spec.js && node scripts/generate.js
```

Then verify output in `docs/` before committing.

## Pantheon API notes

- Spec: `https://api.pantheon.io/docs/swagger.json` (Swagger 2.0)
- Auth: `POST /v0/authorize/machine-token` → Bearer token
- Async pattern: write ops return a workflow ID; poll `GET /v0/sites/{id}/workflows/{id}`
- 79 paths, 119 schema definitions, tags: auth / organizations / sites / users
